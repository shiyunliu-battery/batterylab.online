# Preconditioning Test — Handbook Summary

## Primary Planning Role

- Treat this chapter as the mandatory first step before any formal characterisation campaign.
- The gate is capacity stability, not a fixed number of cycles.
- Do not present this test as forming or conditioning the cell beyond the chapter's intent.

## Locked Core Elements

- Minimum 3 charge–discharge cycles at the standard-cycle rate (manufacturer-defined).
- Capacity stability gate: difference between successive discharge capacities must be ≤ 2%.
- Maximum 10 discharges; flag any cell that does not stabilise within this limit.
- Temperature must be ambient (25 ± 2 °C); no exact temperature prescribed, but must be stable.
- 30 min rest between charge and discharge steps.

## Key Source Facts (p. 15)

- The test verifies capacity is stable because early-cycle capacity can increase or deteriorate.
- The application is solely to give the rest of the test programme a stable starting point.
- Equipment: battery cell tester with sufficient current for the prescribed C-rate, and temperature sensors.

## Planner Notes

- This method does **not** generate characterisation data for reporting; it is a go/no-go gate.
- If the cell does not pass within 10 cycles, do not proceed to other methods without a reviewed decision.
- Capacity values from preconditioning should not be used as the reference capacity for RPTs.
